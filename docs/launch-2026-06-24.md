## 六、回滚方案

### 6.1 软回滚(推荐,影响最小)

如果发现小问题,可以**只回滚部分 commit**:

```bash
# 场景: JS 清理出问题,但 login 清理 OK
git revert 343fa18
# 结果: HEAD 多一个 "Revert ..." commit,3 个被删文件恢复

# 场景: 全部回滚到 8a9e607 之前
git revert 343fa18 53aa2ea 8a9e607
# 或更彻底:
git reset --hard 8a9e607
# 然后强制推送(慎用,影响其他协作者)
```

### 6.2 硬回滚(紧急)

```bash
# 立即回到本次推送前状态
git reset --hard ebfa03c
# 强制推(覆盖远程 main)
git push --force-with-lease origin master:main
```

### 6.3 单文件恢复

如果只需要恢复被删的 3 个文件(其他 commit 内容保留):

```bash
# 本地 backup tag (在本地仓库)
git checkout backup-before-js-cleanup-2026-06-24 -- \
  public/js/admin-product-form.js \
  public/js/admin.js \
  migrate-merge-admin-into-users.js

git commit -m "rollback: restore 3 deleted JS files"
```

**注意**: `backup-before-js-cleanup-2026-06-24` tag **只在本地仓库**,服务器上**没有**。如果服务器需要这个 tag,先在本地推送:

```bash
git push origin backup-before-js-cleanup-2026-06-24
# 然后服务器上:
git fetch --tags
git checkout backup-before-js-cleanup-2026-06-24 -- ...
```

| 数据库连接 | < 50 active | > 80 → 检查 MySQL 池(看 memory `feedback_mysql_keepalive.md`) |

**错误日志关键词**:
- `TypeError` → JS 端问题
- `ECONNRESET` → MySQL 连接问题(参考 memory)
- `Cannot find module` → 依赖问题
- `EADDRINUSE` → 端口被占,先 taskkill zombie(参考 memory `feedback_zombie_node_processes.md`)
